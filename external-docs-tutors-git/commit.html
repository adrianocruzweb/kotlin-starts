Convenções de Mensagens de Commit

No Git, é comum seguir convenções ao escrever mensagens de commit. Essas convenções ajudam a categorizar e entender as mudanças feitas no código.

Exemplos de Convenções:

git commit -m "feat: Adiciona funcionalidade de login"

git commit -m "fix: Corrige erro de validação no formulário"

git commit -m "refactor: Simplifica lógica de autenticação"

git commit -m "chore: Atualiza dependências"

git commit -m "docs: Atualiza documentação do API"

git commit -m "style: Corrige indentação no arquivo"

Escolha uma abordagem que faça sentido para sua equipe e mantenha a consistência ao longo do projeto.

Para mais informações, consulte as Conventional Commits.
